How the JavaScript Event Loop Works

The Event Loop is JavaScript's mechanism for handling asynchronous operations. It consists of:

// Example of different task types
console.log("Sync task"); // Executes immediately

// Microtasks (High Priority)
Promise.resolve().then(() => console.log("Microtask"));

// Macrotasks (Lower Priority)
setTimeout(() => console.log("Macrotask"), 0);
button.addEventListener('click', () => console.log("Click handler"));

Execution Flow:

  1. Synchronous code executes immediately in the Call Stack
  2. Async operations are handed off to Web APIs
  3. When Web APIs complete, callbacks go to respective queues
  4. Event Loop checks if Call Stack is empty
  5. Microtasks (Promises) are processed first, until the queue is empty
  6. Then one Macrotask (setTimeout, events) is processed
  7. Repeat from step 4

Priority Order:

  • 1. Synchronous code (Call Stack)
  • 2. Microtasks (Promise callbacks, queueMicrotask)
  • 3. Macrotasks (setTimeout, setInterval, I/O, UI events)
